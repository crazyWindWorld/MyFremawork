# 2026-05-03

## 游戏服务器架构重设计

### 完成内容
- 将原 .NET Framework 4.7.2 的 TestTcpServer 项目重构为 .NET 8 游戏服务器框架
- 新增 GameServer.csproj，保留原 Proto/ 文件夹的 protobuf 生成文件
- 架构分三层：Gateway（网络）→ Routing（路由）→ Data（数据）

### 新建文件清单
- `GameServer.csproj` - .NET 8 项目文件
- `ProtoCmds.cs` - 协议命令 ID 常量（从旧 ProtoCmds.cs 迁移并扩展）
- `Core/Protocol/Packet.cs` - 网络帧结构
- `Core/Protocol/FrameCodec.cs` - 帧编解码（ArrayPool + BinaryPrimitives，零拷贝）
- `Core/Net/GameSession.cs` - 单连接会话，双 Channel（inbound/outbound）
- `Core/Net/GatewayServer.cs` - TCP 监听 + Session 池 + 广播
- `Core/Routing/MsgHandlerAttribute.cs` - Handler 注册 Attribute
- `Core/Routing/HandlerBase.cs` - 所有 Handler 的基类
- `Core/Routing/MessageRouter.cs` - 反射扫描路由，每条消息独立 DI Scope
- `Core/Routing/HandlerServiceExtensions.cs` - DI 注册扩展
- `Core/Context/PlayerStateCache.cs` - 在线玩家内存缓存
- `Data/Abstractions/IEntity.cs` - 实体标记接口
- `Data/Abstractions/IRepository.cs` - 泛型仓储接口
- `Data/Abstractions/IUnitOfWork.cs` - 工作单元接口
- `Data/InMemory/InMemoryRepository.cs` - 内存仓储（开发/测试用）
- `Data/InMemory/InMemoryUnitOfWork.cs` - InMemory UoW
- `Data/InMemory/InMemoryDataExtensions.cs` - DI 注册扩展
- `Handlers/LoginHandler.cs` - 登录 Handler（使用 LoginPB.LoginReq.Account/Pwd）
- `Handlers/PingHandler.cs` - 心跳 Handler
- `Services/GatewayHostedService.cs` - IHostedService 包装 GatewayServer
- `Services/HeartbeatService.cs` - BackgroundService 心跳检测
- `Program.cs` - 启动入口（Generic Host + DI + Serilog）
- `appsettings.json` / `appsettings.Development.json` - 配置文件

### 关键设计决策
- Proto 字段名：LoginReq 使用 `.Account` 和 `.Pwd`（非 Username/Password）
- 每条消息独立 DI Scope（支持 Scoped 服务如 DbContext）
- IRepository 抽象不绑定具体 DB，后续接入 MySQL/SQLite/MongoDB 只需实现接口
- Channel<Packet> 解耦收发，避免发送阻塞接收

## 登录模块 & 心跳模块 完善实现

### 完成内容
- 完整登录流程：参数校验 → 账号查询 → 自动注册 → 封禁检测 → 密码校验 → 重复登录踢旧连接 → 注册在线缓存 → 返回 LoginRsp
- 完整心跳流程：PingHandler 更新 MarkActive() → HeartbeatService 定时遍历所有 Session → 超时踢人
- 登录校验中间件：MsgHandlerAttribute 增加 RequireLogin 属性，MessageRouter 分发前校验登录状态

### 修改文件清单
- `Core/Net/GameSession.cs` - 增加 LastActiveAt/LoginTime/IsLoggedIn/MarkActive()/MarkLogin()/MarkLogout()（线程安全 Volatile）
- `Core/Net/GatewayServer.cs` - 注入 PlayerStateCache，断连时从缓存注销；新增 GetAllSessions()/GetSession() 方法
- `Core/Context/PlayerStateCache.cs` - 重写：增加 TryKickAsync()（发 LogoutPush + 踢人）、GetAllPlayerIds()、Unregister(long)
- `Core/Routing/MsgHandlerAttribute.cs` - 增加 RequireLogin 属性（默认 true）
- `Core/Routing/MessageRouter.cs` - _registry 改为存储 (Type, bool)，分发前校验 RequireLogin
- `Handlers/LoginHandler.cs` - 完整重写：10步登录流程，注入 PlayerStateCache，支持自动注册和封禁检测
- `Handlers/PingHandler.cs` - 增加 session.MarkActive() 调用
- `Handlers/LogoutHandler.cs` - 新建：主动登出处理
- `Services/HeartbeatService.cs` - 完整实现超时踢人逻辑（从 GatewayServer 遍历所有 Session，检查 LastActiveAt）
- `ProtoCmds.cs` - 新增 LogoutReq=10008, LogoutRsp=10009, KickPush=10012
- `Program.cs` - SeedData 增加封禁测试账号

### PlayerEntity 新增字段
- Nickname: 昵称
- LastLoginAt: 最后登录时间
- IsBanned: 是否封禁
- BanExpireTime: 封禁到期时间（null=永久）

### 关键设计
- GameSession.MarkActive() 使用 Volatile.Read/Write 保证线程安全
- LoginHandler 标记 RequireLogin=false，PingHandler 标记 RequireLogin=false，其余默认 true
- 踢人流程：PlayerStateCache.TryKickAsync() → 发 LogoutPush → session.MarkLogout() → session.Close()
- 断连清理：GatewayServer.HandleSessionAsync finally 块中调用 _playerCache.Unregister()
